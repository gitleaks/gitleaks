# cvscan

Cloudvisor Security Scanner — secrets & IaC misconfiguration scanning for customer repositories.

Built on [gitleaks](https://github.com/gitleaks/gitleaks) (secrets detection) and [trivy](https://github.com/aquasecurity/trivy) (IaC scanning).

## What it does

- **Secrets scanning**: Detects leaked credentials, API keys, and tokens in git history and working tree
- **IaC scanning**: Detects Terraform and CloudFormation misconfigurations
- **HTML report**: Generates a local report with findings grouped by severity
- **Findings submission**: Optionally sends redacted metadata to Cloudvisor backend (no source code transmitted)

## Installation

### Homebrew (recommended)

```bash
brew tap devisory-engineering/cvscan && brew install cvscan
```

### Direct download

Download the latest release for your platform from the [GitHub Releases](../../releases) page.

> **macOS Gatekeeper note:** On macOS, downloaded binaries trigger a "malware" warning. After extracting, run `xattr -cr cvscan` to clear the quarantine flag.

### Build from source

```bash
go build -o cvscan ./cmd/cvscan/
```

## Usage

### Quick start

```bash
# Scan a folder of repositories
cvscan /path/to/repos

# Scan a single repository
cvscan /path/to/my-app

# Scan only for secrets (skip IaC)
cvscan --scanners secrets /path/to/repos

# Custom report path
cvscan -o my-report.html /path/to/repos
```

No flags required beyond the path. Scanners default to both (secrets + IaC). HTML report auto-opens in your browser.

### Interactive mode (TUI)

```bash
cvscan
```

Launches the interactive terminal UI that guides you through scanner selection, folder picking, and optional submission.

### Scan + submit to Cloudvisor

```bash
cvscan --id eng_7kx9m2p4q8 --token tok_a3f8x9k2m1 /path/to/repos
```

Scans, generates report, and submits findings to the Cloudvisor backend.

### Submit after the fact

```bash
cvscan submit --id eng_7kx9m2p4q8 --token tok_a3f8x9k2m1
cvscan submit --id eng_7kx9m2p4q8 --token tok_a3f8x9k2m1 --file ./results.json
```

Reads `.cvscan-results.json` from the current directory (or `--file` path) and submits without re-scanning.

### Flags

```
--id            Engagement ID (eng_xxx) — required for submission
--token         Submission token (tok_xxx) — required with --id
--scanners      Comma-separated: secrets,iac (default: both)
--output, -o    HTML report path (default: cvscan-report.html)
```

## How submission works

Submission is **opt-in**. By default, cvscan runs locally with no network calls.

To submit findings to Cloudvisor, both flags are required:

- **`--id eng_xxx`** — Engagement ID provided by Cloudvisor during kickoff
- **`--token tok_xxx`** — Submission token generated by a Cloudvisor engineer via `/cvscan-token` in Slack (single-use, 7-day expiry)

Each scan produces two local files:
- `cvscan-report.html` — Human-readable report
- `.cvscan-results.json` — Machine-readable findings (enables `cvscan submit`)
